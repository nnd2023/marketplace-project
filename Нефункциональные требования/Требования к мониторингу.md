## 4. OBS. Требования к мониторингу и наблюдаемости (Monitoring / Observability)

| ID | Сценарий | Условия / Событие | Ожидаемый результат | Критерий приемки |
| :--- | :--- | :--- | :--- | :--- |
| **OBS-001** | Сбор и агрегация логов | Любое событие в системе | Логи доступны для поиска и анализа в централизованном хранилище | Все сервисы пишут в stdout/structured JSON, сбор через Fluentd/Vector в ELK/PLG, хранение ≥ 30 дней |
| **OBS-002** | Метрики производительности | Непрерывная работа сервисов | Доступны метрики для анализа здоровья и производительности | Сбор метрик (Prometheus) с интервалом ≤ 15 сек: RPS, latency (p50/p95/p99), error rate, CPU, memory |
| **OBS-003** | Оповещение об инцидентах | Превышение порога ошибок или latency | Ответственная команда получает уведомление | Правила алертинга в Alertmanager, уведомление в PagerDuty/Slack ≤ 1 минуты после детекта |
| **OBS-004** | Аудит действий пользователей | Изменение критических данных (цена, статус заказа) | Действие зафиксировано в неизменяемом логе аудита | Запись в аудит-лог: кто, что, когда, до/после; лог защищен от модификации, хранение ≥ 1 год |
