## 2. PRF. Требования к производительности (Performance)

| ID | Сценарий | Условия / Нагрузка | Ожидаемый результат | Критерий приемки |
| :--- | :--- | :--- | :--- | :--- |
| **PRF-001** | Поиск товаров в каталоге | 1000 RPS, 95-й перцентиль | Пользователь получает результаты поиска | Время ответа ≤ 200 мс (p95), ≤ 500 мс (p99) |
| **PRF-002** | Оформление заказа (Checkout) | Пиковая нагрузка, 500 одновременных чекаутов | Заказ успешно создается, оплата инициируется | End-to-end время ≤ 2 сек, без таймаутов и потерь |
| **PRF-003** | Загрузка карточки товара | Мобильное устройство, 4G-сеть | Страница полностью отрисована и интерактивна | LCP ≤ 2.5 сек, FID ≤ 100 мс, CLS ≤ 0.1 |
| **PRF-004** | Пиковая нагрузка (распродажа) | Увеличение трафика в 10× от базового | Система обрабатывает запросы без критических ошибок | Ошибки 5xx ≤ 0.1%, время ответа критических путей деградирует не более чем на 50% |
| **PRF-005** | Работа с кэшем | Высокая частота чтения каталога | Данные отдаются из кэша, нагрузка на БД снижена | Hit-rate Redis ≥ 80%, время ответа из кэша ≤ 10 мс |
