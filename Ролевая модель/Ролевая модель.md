# Роли в системе маркетплейса

Документ описывает роли участников системы маркетплейса, их функциональное назначение и взаимодействие с системой.

| Роль | Описание | Сценарии использования | Участники |
|------|----------|------------------------|-----------|
| ROLE_CUSTOMER - Покупатель | Конечный потребитель товаров. Ищет, просматривает, добавляет в корзину и приобретает товары. Осуществляет возвраты, оставляет отзывы, управляет личным кабинетом (история заказов, адреса доставки, избранное). | • UC-CUST-001 Поиск и фильтрация товаров<br>• UC-CUST-002 Оформление заказа<br>• UC-CUST-003 Отслеживание доставки | • Физические лица — конечные покупатели |
| ROLE_SELLER - Продавец | Юридическое лицо или ИП, размещающее товары на маркетплейсе. Управляет каталогом товаров (добавление/редактирование/удаление SKU), ценами, остатками на складе. Формирует отгрузочные документы, отслеживает статусы заказов, получает отчёты по продажам и выплатам. | • UC-SELL-001 Управление каталогом товаров<br>• UC-SELL-002 Обновление остатков | • Поставщики-партнёры (3PL)<br>• Бренды-производители |
| ROLE_MARKETPLACE_ADMIN - Администратор маркетплейса | Суперпользователь платформы. Управляет глобальными настройками системы: валюты, страны, налоговые правила, комиссии маркетплейса. Назначает администраторов подразделений. Имеет доступ ко всем данным системы без ограничений. | • UC-ADMIN-001 Настройка комиссий<br>• UC-ADMIN-002 Управление регионами доставки | • Технические администраторы маркетплейса |
| ROLE_WAREHOUSE_OPERATOR - Оператор склада | Сотрудник склада маркетплейса или партнёрского склада. Приёмка товаров от продавцов, размещение на полках, комплектация заказов, упаковка, передача курьеру. Работает с терминалом сборки и сканерами штрихкодов. | • UC-WH-001 Приёмка товара от продавца<br>• UC-WH-002 Комплектация заказа | • Операторы приёмки<br>• Комплектовщики |
| ROLE_COURIER - Курьер доставки | Курьер службы доставки. Получает заказы через мобильное приложение, доставляет покупателю, получает подтверждение получения (подпись/фото), возвращает невыкупленные заказы на склад. | • UC-COUR-001 Приём маршрута доставки<br>• UC-COUR-002 Доставка заказа покупателю | • Штатные курьеры маркетплейса |
| ROLE_SUPPORT - Поддержка | Сотрудник службы поддержки. Обрабатывает обращения покупателей и продавцов через тикет-систему: вопросы по заказам, возвратам, спорам, техническим проблемам. Имеет расширенные права на просмотр данных заказа и изменение статусов в рамках регламента. | • UC-SUP-001 Просмотр обращения клиента<br>• UC-SUP-002 Изменение статуса заказа | • Агенты первой линии поддержки |
| ROLE_MODERATOR - Модератор контента | Модератор контента. Проверяет товары, изображения, описания и отзывы на соответствие правилам площадки (цензура, достоверность, отсутствие запрещённых товаров). Принимает решения об отклонении/одобрении контента или блокировке. | • UC-MOD-001 Модерация карточки товара | • Модераторы контента отдела качества |
| ROLE_CATEGORY_MANAGER - Менеджер категорий | Эксперт по товарной категории. Формирует и поддерживает структуру категорий и атрибутов. Анализирует рынок, конкурентов, спрос. Утверждает новые категории и атрибуты для продавцов. | • UC-CATMGR-001 Управление категориями и атрибутами | • Менеджеры категорий (одежда, электроника, быт и т.д.) |
| ROLE_SELLER_MANAGER - Менеджер по работе с продавцами | Менеджер по работе с ключевыми продавцами. Сопровождает крупных поставщиков: помогает с интеграцией, решает операционные проблемы, согласовывает условия сотрудничества, проводит обучение. | • UC-SELLMGR-001 Онбординг нового продавца | • Аккаунт-менеджеры отдела продаж |
| ROLE_FINANCE_OPERATOR - Финансовый оператор | Финансовый оператор. Обрабатывает выплаты продавцам, сверяет расчёты, формирует акты сверки, обрабатывает возвраты средств покупателям. Контролирует корректность начисления комиссий. | • UC-FIN-001 Обработка выплаты продавцу | • Бухгалтерия маркетплейса |
| ROLE_MARKETER - Маркетолог | Маркетолог платформы. Настраивает и управляет рекламными кампаниями, промо-акциями, купонами, скидками. Анализирует эффективность маркетинговых активностей. | • UC-MKT-001 Создание промо-акции | • Маркетинговый отдел |
| ROLE_ANALYST - Аналитик | Бизнес-аналитик / аналитик данных. Формирует отчёты и дашборды по ключевым метрикам: GMV, конверсия, удержание, логистика. Проводит глубокий анализ для принятия управленческих решений. | • UC-ANAL-001 Формирование бизнес-отчёта | • Отдел бизнес-аналитики |
| ROLE_SYSTEM_ADMIN - Системный администратор | Системный администратор ИТ-инфраструктуры. Управляет доступом пользователей, настраивает интеграции, мониторит работоспособность системы, управляет ролями и правами в административной панели. | • UC-SYSADM-001 Управление пользователями и ролями | • ИТ-администрация маркетплейса |
| ROLE_AUDITOR - Аудитор | Внутренний или внешний аудитор. Проводит выборочные проверки операций системы: корректность расчётов, соблюдение регламентов, безопасность данных. Имеет права «только для чтения» с возможностью экспорта данных для анализа. | • UC-AUD-001 Выборочная проверка операций | • Внутренний аудит компании |

# Примечания

1. **Наследование прав**: Некоторые роли могут включать права других ролей (например, ROLE_MARKETPLACE_ADMIN включает права ROLE_CATEGORY_MANAGER и ROLE_MODERATOR в режиме ручного вмешательства).
2. **Мультиаккаунтность**: Один физический пользователь может обладать несколькими ролями (например, сотрудник маркетплейса может одновременно иметь ROLE_CATEGORY_MANAGER и ROLE_ANALYST).
3. **Ссылки на сценарии**: Сценарии использования (UC-XXX-XXX) описываются в отдельном документе.
4. **Расширяемость**: Матрица ролей может быть расширена новыми ролями по мере развития функционала системы.
