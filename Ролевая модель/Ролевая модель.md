# Примечания

1. **Наследование прав**: Некоторые роли могут включать права других ролей (например, ROLE_MARKETPLACE_ADMIN включает права ROLE_CATEGORY_MANAGER и ROLE_MODERATOR в режиме ручного вмешательства).
2. **Мультиаккаунтность**: Один физический пользователь может обладать несколькими ролями (например, сотрудник маркетплейса может одновременно иметь ROLE_CATEGORY_MANAGER и ROLE_ANALYST).
3. **Ссылки на сценарии**: Сценарии использования (UC-XXX-XXX) описываются в отдельном документе.
4. **Расширяемость**: Матрица ролей может быть расширена новыми ролями по мере развития функционала системы.
