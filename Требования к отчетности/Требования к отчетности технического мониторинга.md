# Требования к отчетности технического мониторинга

Настоящий документ описывает требования к отчету технического мониторинга (UC-031).

## Цель
Предоставление технической команде и руководству информации о состоянии инфраструктуры и бизнес-процессов для своевременного реагирования на инциденты.

### Источники данных (Domain Model)
Отчет формируются на основе событийной модели и агрегированных метрик, хранящихся в доменной модели данных.
Агрегаты и сущности:

*   `SystemMetric`: Базовая метрика (CPU, Memory, Disk).
*   `ServiceHealth`: Статус микросервиса (UP/DOWN/Degraded).
*   `BusinessTransaction`: Ключевые бизнес-события (Создание заказа, Оплата).
*   `ErrorLog`: Записи об исключениях в приложении.

### Структура отчета "SLA Dashboard" (Real-time)

| Поле | Тип | Описание | Источник (Entity.Field) |
| :--- | :--- | :--- | :--- |
| `report_id` | UUID | Уникальный идентификатор среза | `SystemMetric.id` |
| `timestamp` | DateTime | Время среза | `SystemMetric.created_at` |
| `service_name` | String | Имя микросервиса | `ServiceHealth.name` |
| `availability_pct` | Decimal | Процент доступности за период | `AggregatedMetric.availability` |
| `avg_latency_ms` | Integer | Средняя задержка ответа (p95) | `AggregatedMetric.latency_p95` |
| `error_rate_pct` | Decimal | Процент ошибок (5xx) | `ErrorLog.count / RequestLog.count` |
| `status` | Enum | Статус SLA (OK/WARNING/CRITICAL) | Calculated |
